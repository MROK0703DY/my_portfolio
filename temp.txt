Лучшее решение в вашей ситуации — уйти от использования двух разных файлов и объединить их в единую логику с защитой от ошибок. Конфликты возникают из-за того, что один скрипт «обгоняет» другой, а разные браузеры по-разному обрабатывают скорость загрузки файлов.
Вот архитектура, которая будет работать стабильно везде:
## 1. Объедините логику в один файл
Создайте один файл, например app.js, и удалите подключение старых. Внутри мы будем использовать Централизованное управление.

// app.js
async function initApp() {
    try {
        // 1. Загружаем конфиг
        const response = await fetch('config.json');
        if (!response.ok) throw new Error('Ошибка загрузки конфига');
        const config = await response.json();

        // 2. Вызываем функцию заполнения ссылок
        setupLinks(config);

        // 3. Тут можно вызвать другие функции, которым нужен конфиг
        console.log("Приложение инициализировано");

    } catch (error) {
        console.error("Критическая ошибка:", error);
    }
}
function setupLinks(config) {
    // Список ID и соответствующих ключей в JSON
    const mapping = {
        'link-tg': config.telegram,
        'link-yt': config.youtube,
        'link-site': config.website,
        'link-contact': config.email,
        'link-other': config.other // Та самая 5-я ссылка
    };

    // Проходим циклом по всем элементам
    for (const [id, url] of Object.entries(mapping)) {
        const element = document.getElementById(id);
        
        if (element) {
            // Исправляем undefined: если в конфиге нет ссылки, ставим заглушку '#'
            element.href = url || '#'; 
            
            if (!url) {
                console.warn(`Предупреждение: ссылка для ID "${id}" не найдена в JSON`);
            }
        }
    }
}
// Запуск только после полной загрузки DOM (защита для старых браузеров)
document.addEventListener('DOMContentLoaded', initApp);

------------------------------
## 2. Почему это решит проблемы:

* Нет гонки данных: Скрипт не начнет искать ссылки в HTML, пока fetch не вернет данные из JSON.
* Исправление undefined: Строка url || '#' проверяет: если ссылки в JSON нет, сайт не сломается, а просто поставит пустую ссылку.
* Кроссбраузерность: Использование DOMContentLoaded гарантирует, что скрипт увидит все теги <a>, даже если браузер медленно отрисовывает страницу.
* Чистый HTML: Вам нужно подключить только один скрипт в самом конце перед тегом </body>.

## 3. Как теперь должен выглядеть ваш HTML
Убедитесь, что у ссылок есть правильные id, которые вы указали в объекте mapping в коде выше.

<!-- Соцсети -->
<a id="link-tg" href="#">Telegram</a>
<a id="link-yt" href="#">YouTube</a>
<a id="link-site" href="#">Сайт</a>
<!-- Подключаем только ОДИН файл -->
<script src="js/app.js"></script>

## 🛠 Как проверить, если что-то не работает:

   1. Нажмите F12 -> вкладка Network (Сеть). Проверьте, что файл config.json горит зеленым (статус 200).
   2. Вкладка Console (Консоль). Если там есть надпись «Предупреждение: ссылка для ID... не найдена», значит в вашем config.json ключ написан иначе, чем в коде.
